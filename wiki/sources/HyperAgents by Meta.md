## 요약

Meta와 UBC의 새 논문에서 발표한 **HyperAgents** (DGM-H)를 소개하는 글. 자기 참조적(self-referential) AI 에이전트가 태스크 해결 행동뿐 아니라, 미래의 개선을 생성하는 메커니즘 자체를 수정한다는 개념. 핵심 발견: 에이전트를 자유롭게 자기 개선시키면 개발자가 수동으로 만드는 것과 동일한 하네스 컴포넌트를 독자적으로 재발명한다.

## 핵심 구조: DGM-Hyperagents

하나의 하이퍼에이전트는 두 가지로 구성:
- **태스크 에이전트**: 주어진 태스크 해결
- **메타 에이전트**: 태스크 에이전트와 **자기 자신**을 수정

핵심 키워드: **메타인지적 자기 수정 (metacognitive self-modification)**

진화 루프:
1. 기본 에이전트로 시작
2. 메타 에이전트가 코드 분석 + 과거 성능 기반으로 수정본 생성
3. 평가 → 성능 향상 시 아카이브 추가
4. 아카이브에서 선택 → 반복

## 자연 발생하는 하네스 컴포넌트

코딩, 논문 리뷰, 로봇공학, 수학 채점 등 다양한 도메인에서 에이전트가 **독립적으로** 발명한 것들:

| 컴포넌트 | 설명 |
|---------|------|
| **영속 메모리** | 과거 결과, 성능 트렌드, 합성 인사이트 저장 |
| **성능 추적** | 이동 평균, 세대별 통계, 도메인별 점수 이력 |
| **다단계 평가 파이프라인** | 체크리스트, 결정 규칙, 명확한 기준 |
| **임계값 기반 결정 프로토콜** | 수락/거절 비율, 신뢰도 수준 |
| **도메인 지식 베이스** | 제약 조건, 유효 상태 변수, 발견적 지식 |
| **재시도 및 자기 수정** | 회귀 감지 후 수정 |

이는 개발자가 **수동으로** 하네스에 구축하는 것과 정확히 같은 컴포넌트들이다.

## 시사점

> 하네스는 개발자 편의 기능이 아니라 에이전틱 시스템의 **수렴 아키텍처**다.

- **유니버설 에이전트**: 하네스 안에서 실행되는 적응형 실행자
- **하이퍼에이전트**: 하네스를 *생산*하는 존재

개발자의 역할 변화: 하네스를 직접 구축 → 에이전트가 효과적인 하네스를 진화시킬 수 있는 **초기 조건을 설계**

## 더 큰 그림: 수렴하는 트렌드

| 작업 | 방향 |
|------|------|
| 하네스 엔지니어링 | 에이전트 주변에 6가지 컴포넌트 구축 |
| Copilot → Codex | 인간 작성 코드 → 에이전트 위임 코드 |
| 유니버설 에이전트 | 코딩 능력이 범용성 만들어냄 |
| **HyperAgents** | 에이전트가 자기 하네스를 직접 구축 |

에이전트는 **인프라 소비자에서 인프라 생산자로** 이동 중.
