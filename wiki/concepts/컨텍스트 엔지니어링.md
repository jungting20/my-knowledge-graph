## 정의

컨텍스트 엔지니어링(Context Engineering)이란 AI 에이전트에게 **어떤 정보를 보여주고, 어떤 정보를 숨기며, 무엇을 다음 단계로 넘길지**를 구조적으로 다루는 방식이다.

## 진화의 흐름

```
프롬프트 엔지니어링 → 컨텍스트 엔지니어링 → 하네스 엔지니어링 → 에이전틱 엔지니어링
```

각 단계가 이전을 대체하는 것이 아니라, 모두 중요해진 것이다.

## 핵심 원칙

> 컨텍스트는 많이 넣는다고 좋아지지 않는다. **압축력**의 문제다.

### 컨텍스트 계층 분리

| 계층 | 내용 | 관리 방법 |
|------|------|---------|
| **지속 규칙** | CLAUDE.md 등 항상 적용되는 규칙 | 짧게 핵심만 |
| **작업 상태** | 현재 진행 중인 작업의 중간 산출물 | session state로만 유지 |
| **근거 자료** | 답변 생성에 필요한 증거 | 요약된 snippet + 문서 ID |
| **임시 정보** | 이번 턴에만 필요한 정보 | 사용 후 제거 |

### Specialist에게 전달 시

전체 대화가 아니라 **요약된 작업 카드만** 넘길 것

## Claude Code의 컨텍스트 전략

1. **계층적 메모리 로딩**: 프로젝트 → 사용자 → 글로벌 순서
2. **자동 압축**: 컨텍스트 과다 시 이전 메시지 요약
3. **Progressive Disclosure**: 에이전트가 필요할 때 점진적으로 컨텍스트 탐색
4. **서브에이전트 격리**: 특정 태스크의 컨텍스트 오염을 서브에이전트에 격리

## CLAUDE.md의 역할

단순한 문서가 아니라 **에이전트의 운영 매뉴얼** (Boris Cherny 표현: "두 번째 교대의 운영 매뉴얼")

- `/loop` 실행 시 매 사이클마다 재독
- 에이전트의 규칙, 컨벤션, 아키텍처 결정이 담김
- 품질이 높을수록 루프 출력 품질도 높아짐
