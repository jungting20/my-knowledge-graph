## 정의

에이전트 하네스(Agent Harness)란 AI 에이전트가 작동하는 방식을 관장하는 소프트웨어 시스템이다. 도구, 메모리, 재시도, 컨텍스트 엔지니어링, 검증을 관리하여 모델이 추론에 집중할 수 있게 한다.

> "A harness is the software system that governs how an AI agent operates." — Cobus Greyling

## 하네스의 6가지 핵심 컴포넌트

| 컴포넌트 | 설명 |
|---------|------|
| **Tool Integration** | 도구 등록 및 실행 |
| **Memory & State** | 단계 간 결과 영속화 |
| **Context Engineering** | 프롬프트 동적 조립 |
| **Planning** | 복잡한 태스크를 단계로 분해 |
| **Verification** | 규칙에 따른 출력 검증 |
| **Modularity** | 컴포넌트 독립적 토글 |

## 하네스가 중요한 이유

에이전트의 완성도는 "얼마나 똑똑한 모델을 쓰느냐"보다 **"어떤 루프와 상태 위에서 움직이는가"** 에서 갈린다.

- 동일한 AI 모델도 하네스에 따라 완전히 다른 결과를 낸다
- [[HyperAgents by Meta]] 연구: 에이전트를 자유롭게 자기 개선시키면 동일한 하네스 컴포넌트를 **독자적으로 재발명**한다 → 수렴 아키텍처임을 의미

## Claude Code의 하네스 구조 (유출 기반)

```
submitMessage() → System Prompt 결합 → 42개 도구 동적 배치
→ 컨텍스트 압축 (과다 시) → 메모리 계층적 로딩
→ tool_use 감지 → 권한 정책 평가 → 실행 → 결과 재주입
→ 반복 (max_turns 도달 또는 final 응답까지)
```

## 하네스 vs 모델

| 비교 | 내용 |
|------|------|
| **모델** | 추론 능력 제공, 세션 간 기억 없음 |
| **하네스** | 컨텍스트 관리, 도구 실행, 상태 유지, 안전장치 |

개발자 역할의 변화:
- 이전: 하네스를 **직접 구축**
- 미래 ([[HyperAgents]]): 에이전트가 효과적인 하네스를 진화시킬 수 있는 **초기 조건을 설계**
