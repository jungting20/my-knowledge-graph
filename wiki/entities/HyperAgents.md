## 개요

HyperAgents(DGM-Hyperagents)는 Meta와 UBC(브리티시컬럼비아 대학)의 공동 연구 프로젝트. 자기 참조적(self-referential) AI 에이전트에 관한 논문으로, 에이전트가 태스크 해결 행동뿐 아니라 개선 메커니즘 자체를 수정한다.

## GitHub

- [facebookresearch/HyperAgents](https://github.com/facebookresearch/Hyperagents)
- arxiv: https://arxiv.org/abs/2603.19461

## 핵심 구조

단일 편집 가능한 프로그램으로 두 가지 포함:
- **태스크 에이전트**: 주어진 태스크 해결
- **메타 에이전트**: 태스크 에이전트와 **자기 자신** 수정

**메타인지적 자기 수정(metacognitive self-modification)**이 핵심.

## 진화 루프

1. 기본 에이전트 시작 (LLM 단일 호출, 도구/메모리/플래닝 없음)
2. 메타 에이전트가 코드 분석 + 과거 성능 기반으로 수정본 생성
3. 평가 → 성능 향상 시 아카이브 추가
4. 아카이브에서 선택 → 수백 번 반복

## 독립 발명한 하네스 컴포넌트

코딩, 논문 리뷰, 로봇공학, 수학 채점 등 다양한 도메인에서:

| 컴포넌트 | 관찰 |
|---------|------|
| 영속 메모리 | 인과적 가설, 세대별 성능, 성공 전략 저장 |
| 성능 추적 | 이동 평균, 도메인별 점수 이력 |
| 다단계 평가 파이프라인 | 체크리스트, 결정 규칙, 명확한 기준 |
| 임계값 기반 결정 프로토콜 | 수락/거절 비율, 신뢰도 수준 |
| 도메인 지식 베이스 | 제약 조건, 유효 변수, 발견적 지식 |
| 재시도 및 자기 수정 | 회귀 감지 후 수정 |

이는 [[에이전트 하네스]]의 6가지 컴포넌트와 **동일**하다 → 수렴 아키텍처 증거.

## 시사점

- 하네스는 개발자 편의 기능이 아닌 **에이전틱 시스템의 수렴 아키텍처**
- 개발자 역할: 하네스 직접 구축 → 에이전트가 하네스를 진화시킬 **초기 조건 설계**
